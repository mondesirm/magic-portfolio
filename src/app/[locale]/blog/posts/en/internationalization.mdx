<details>
<summary><strong>What is internationalization?</strong></summary>

Internationalization (often abbreviated as i18n) is the process of designing and developing a product, application, or website in a way that allows it to be easily adapted for different languages and regions without requiring significant changes to the underlying codebase. The term "i18n" comes from the fact that there are 18 letters between the first letter "i" and the last letter "n" in the word "internationalization."

</details>

---

## Why and how I implemented internationalization

I use both English and French daily, so I wanted to make my portfolio accessible to a wider audience. To achieve this, I decided to implement internationalization using the `next-intl` library. This library provides a simple and efficient way to handle translations and locale management in Next.js applications.

I noticed internationalization was dropped on the original template, so I added it back and then merged it with all the custom modifications I had already made to my portfolio.

## Use it as a reference for your own portfolio site

If you want to explore the version of the template I updated and adapted for multilingual support, you can use the [`i18n` branch](https://github.com/mondesirm/magic-portfolio/tree/i18n) as a starting point for your own portfolio.

If you want to show only one language for now, you can either set the `i18n` variable in `src/resources/once-ui.config.ts` to your desired language or simply use the `develop` branch which is my modified version of the template without internationalization. The former will allow you to focus on a single language while still having the option to add more languages in the future.
